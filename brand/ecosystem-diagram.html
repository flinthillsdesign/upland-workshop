Shared Auth Pattern

Scheduler, ODIN, and Inquiry Hub share a single Turso database (upland-auth) with one users table. All three use the same JWT_SECRET — a token minted by one app is valid in all three. The Upland Website has its own PostgreSQL auth and stays intentionally separate.

Dual Database Pattern

Each Netlify app connects to two Turso databases: the shared auth DB (TURSO_AUTH_URL) and its own app DB (TURSO_URL). Local dev uses SQLite files; production uses Turso HTTP. The client import auto-switches based on URL prefix.

ODIN as Intelligence Layer

ODIN is read-only — it never writes to another app's data. It pulls from Scheduler, Inquiry Hub, Storefront orders, Basecamp, and Toggl, then uses Claude AI to synthesize agendas, briefings, and answers. It also provides an enrichment API that Inquiry Hub calls for context.

Standalone Severability

Every app can survive independently. If ODIN goes down, inquiries still work. If the Scheduler goes offline, ODIN degrades gracefully. The Upland Website is on its own stack entirely (Python, PostgreSQL, Heroku) and doesn't depend on any satellite app.

Convention Over Shared Code

No shared npm package or component library. Instead, all Netlify apps follow the same patterns: esbuild + vanilla TypeScript, hand-rolled router in a single function, ensureSchema() migrations, Postmark email, oxlint/oxfmt. Copy the pattern, not the dependency.

Visual Identity

All apps share: dark gradient background (#161A22 to #0A0D12), lime accent (#A5BC39), Instrument Sans font, rounded-rect app icons on the same dark base. Each icon's content differentiates: sparkles for ODIN, chat bubble for Inquiries, Gantt bars for Scheduler, etc.
